# AGENTS.md

## Проект

- Это расширение VS Code/Cursor для запуска `.http`-файлов JetBrains HTTP Client через установленный `ijhttp`.
- Основная логика находится в `src/ijhttp/ijHttpCliRunner.ts` и `src/utils/selector.ts`.
- Расширение не содержит собственного HTTP-рантайма: выполнение всегда делегируется `ijhttp`.

## Важный контекст

- Запуск одного блока и `Run all` используют временные `.http`-файлы.
- Перед выполнением обычные комментарии должны вырезаться из исполняемого текста запроса.
- История ответов сохраняется в каталог `.response/` рядом с исходным `.http`.
- История хранится как полный дамп ответа в файлах `*.response`, а не как чистый JSON body.
- Для `*.response` есть отдельная подсветка языка `ijhttp-response`.

## Изменения в коде

- Сохраняй текущую архитектуру: парсинг блоков в `Selector`, выполнение и файловую логику в `IjHttpCliRunner`.
- Не добавляй extension tests для реального запуска VS Code, если пользователь явно не попросил.
- Для автотестов предпочитай `vitest`.
- В первую очередь покрывай регрессии в `Selector` и `IjHttpCliRunner`.

## Проверки

- `npm test` — это быстрые unit/regression тесты.
- `npm run test:e2e` — это отдельные e2e-тесты с реальным `ijhttp` и временным локальным сервером.
- После изменений в логике TypeScript запускай `npm test`, если тесты затронуты или могут поймать регрессию.
- Если правка затрагивает интеграцию исполнения запросов, сохранение `.response` или поведение history-комментариев, также запускай `npm run test:e2e`.
- После изменений в расширении проверяй сборку через `npm run webpack`.
- Если меняется упаковка или версия, проверяй итоговый `.vsix`.

## Версия и упаковка

- Если пользователь пишет `подними и упакуй` (`bump`) после правок и не указывает конкретную версию, по умолчанию увеличивай версию на `0.0.1` и затем собирай новый `.vsix`.
- Если пользователь явно указал версию, используй её вместо значения по умолчанию.
- Для изменения версии используй `npm version <version> --no-git-tag-version`.
- Для упаковки используй `npx @vscode/vsce package`.
